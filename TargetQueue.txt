Documentation for Target Queue

Goal: Create a Priority Queue using a doubly linked list. This way one traversal is First and the other traversal is Last.

The TargetQueue should be able to reference the spawnIndex of an enemy or calculate and use a different metric in order to sort the enemies into the correct order.

BEGIN
Capsule should create an instance of the TargetQueue.

EACH FRAME
Capsule should check for all enemies in range and check if they are already in the linked list by checking the spawnIndexValue. If the enemy is not in the linked list, push() the enemy onto the queue.

If TargetQueue ==  First
  If the player selects TargetPriority First
    TargetQueue should sort the enemies in the Queue by lowest Spawn Index to highest SpawnIndex.
    When Capsule asks for Target, it should peek().

If TargetQueue == Last
  If the player selects TargetPriority Last
    TargetQueue should sort the enemies in the Queue by Highest Spawn Index to Lowest SpawnIndex.
    When Capsule asks for Target, it should peek().


We spent a couple of hours working through what our best data structure would be given the constraints of the game. We came to the conclusion that heaps would be the best and the easiest, but we don't think we are allowed to use heaps.

We are proceeding with a doubly linked list because we want to be able to traverse in more than one way.

It is worth noting that there is a vector of enemies that is built by the spawner and can be referenced for adding Enemies to the doubly linked List IF the enemy is in range. Each enemy also has a Unique identifier = spawnIndex.
The FIRST target priority can be made by traversing the DLL in order, and the Last priority cal be made by traversing the DLL in reverse.

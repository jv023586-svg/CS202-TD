Documentation for Target Queue

Goal: Create a Priority Queue using  a linked list.

The TargetQueue should be able to reference the spawnIndex of an enemy or calculate and use a different metric in order to sort the enemies into the correct order.

BEGIN
Capsule should create an instance of the Target Queue

EACH FRAME
Capsule should check for all enemies in range and check if they are already in the linked list by checking the spawnIndexValue. If the enemy is not in the linked list, push() the enemy onto the queue.

If TargetQueue ==  First
  If the player selects TargetPriority First
    Capsule should sort the enemies in the Queue by lowest Spawn Index to highest SpawnIndex.

If TargetQueue == Last
  If the player selects TargetPriority Last
    Capsule should sort the enemies in the Queue by Highest Spawn Index to Lowest SpawnIndex.

If TargetQueue == LowestHP
  If the player selects TargetPriority LowestHP
    Capsule should sort the enemies in the queue by largest health to smallest health.
